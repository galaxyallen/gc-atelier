## 2. Vercel 部署

### 2.1 首次准备

1. 在 [Supabase](https://supabase.com) 创建项目
2. **Database**：Settings → Database → Connection string（URI）→ 用作 `DATABASE_URL`
3. **Storage**：创建 **public** bucket，名称 `uploads`
4. 在 [Vercel](https://vercel.com) 导入 Git 仓库，Root Directory 设为 `gc-atelier`（若仓库根目录不是项目根）

### 2.2 构建配置

Vercel 自动识别 Next.js，默认构建命令：

```bash
npm run build
```

即 `prisma generate && next build`（见 `package.json`）。

**注意：**

- Build 只用 `prisma generate`，**不要** `prisma db push`
- 数据库结构通过 `prisma/migrations/` 管理
- `next.config.mjs` 对 CMS 页面设置 `Cache-Control: no-store`，确保后台更新后前台及时刷新

### 2.3 生产环境变量

参考 `env.vercel.example`，在 Vercel → Project → Settings → Environment Variables → **Production** 配置：

| 变量 | 值 |
|------|-----|
| `DATABASE_URL` | Supabase Postgres URI（带 `?pgbouncer=true` 时用于 serverless） |
| `NEXTAUTH_SECRET` | 随机长字符串 |
| `NEXTAUTH_URL` | `https://gccreativehk.com` |
| `NEXT_PUBLIC_SITE_URL` | `https://gccreativehk.com` |
| `NEXT_PUBLIC_SUPABASE_URL` | `https://xxx.supabase.co` |
| `SUPABASE_SERVICE_KEY` | Supabase **service_role** key（仅服务端，勿暴露到客户端） |
| `STRIPE_SECRET_KEY` | `sk_test_...` 或 `sk_live_...` |
| `STRIPE_WEBHOOK_SECRET` | `whsec_...` |
| `STRIPE_CURRENCY` | `usd` |
| `NEXT_PUBLIC_SHOP_CURRENCY` | `USD` |
| `SETUP_SECRET` | 一次性初始化（用后删除） |

本地校验（填入 Supabase 变量后）：`npm run deploy:check-env`

创建 Storage bucket：`npm run supabase:setup`

### 2.4 部署方式

**Git 自动部署（推荐）：** 连接仓库后推送 `main` 自动构建。

**CLI 手动部署：**

```bash
cd gc-atelier
npx vercel --prod
```

### 2.5 生产数据库初始化

1. 在 Supabase SQL Editor 或本地执行 `prisma migrate deploy` 应用迁移
2. 设置 `SETUP_SECRET`（随机字符串）并重新部署
3. 执行：

```bash
curl -X POST https://gccreativehk.com/api/setup \
  -H "x-setup-secret: 你的SETUP_SECRET"
```

4. 成功后**删除** `SETUP_SECRET` 并再次部署

### 2.6 域名与 DNS

在 Vercel 绑定 `gccreativehk.com` 与 `www.gccreativehk.com`，按 Vercel 控制台提示配置 DNS：

| 类型 | 主机 | 值（示例） |
|------|------|------------|
| A | `@` | `76.76.21.21` |
| CNAME | `www` | `cname.vercel-dns.com` |

启用 HTTPS（Vercel 自动签发证书）。

## 3. Stripe 支付运维

### 3.1 配置步骤

1. [Stripe Dashboard](https://dashboard.stripe.com) → API Keys → Secret key → `STRIPE_SECRET_KEY`
2. 创建 Webhook 端点：

| 项目 | 值 |
|------|-----|
| URL | `https://gccreativehk.com/api/payments/webhook` |
| 事件 | `checkout.session.completed`、`checkout.session.expired` |

3. Signing secret → `STRIPE_WEBHOOK_SECRET`

### 3.2 测试 vs 正式

| 模式 | 密钥 | 测试卡 | 查看付款 |
|------|------|--------|----------|
| 测试 | `sk_test_` | `4242 4242 4242 4242` | Test mode → Payments |
| 正式 | `sk_live_` | 真实卡 | Live mode → Payments |

**常见错误：**

- 测试卡 + Live 密钥 → 支付被拒
- Test Webhook + Live 密钥 → 签名失败
- Test / Live 的 Webhook secret **不能混用**

### 3.3 支付状态排查

订单卡在 PENDING 时检查：

1. Stripe Dashboard → Webhooks → 最近事件是否 200
2. `STRIPE_WEBHOOK_SECRET` 与当前模式一致
3. Webhook URL 可公网访问
4. 事件包含 `checkout.session.completed`

## 5. 故障排查

| 现象 | 原因 | 处理 |
|------|------|------|
| CMS 更新前台不变 | CDN/ISR 缓存 | 确认 `revalidateSitePages()` 被调用；检查 Cache-Control headers |
| 图片上传失败 | Serverless 无磁盘 | 配置 Supabase Storage（`uploads` bucket） |
| `prisma db push` 报错 | 生产应走迁移 | 使用 `prisma migrate deploy` |
| 支付成功订单仍 PENDING | Webhook 问题 | 检查 URL、事件、secret、Test/Live 模式 |
| 构建失败 | Prisma/依赖 | 查看 Vercel build log |
| Supabase 连接超时 | serverless 连接池 | 使用 Supabase Transaction pooler URI |
