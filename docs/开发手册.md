# GC Atelier — 开发手册

面向开发者的本地环境搭建、代码架构与功能实现说明。

> 部署、生产配置与日常运维请参阅 [运维手册](./运维手册.md)。

---

## 1. 项目概览

| 项目 | 说明 |
|------|------|
| 项目名称 | GC Atelier CMS + 电商前台 |
| 代码目录 | `gc-atelier/` |
| 技术栈 | Next.js 14 (App Router)、TypeScript、Tailwind CSS、Prisma 7、NextAuth、Supabase (Postgres + Storage)、Stripe |

### 功能模块

- **前台**：首页、Studio、Services、Projects、Shop、Contact
- **后台 CMS**：内容、商品/项目、订单、询盘、媒体、用户
- **用户系统**：注册/登录、个人资料、订单
- **支付**：Stripe Checkout（USD）
- **认证分离**：管理员 `/admin/login` 与前台用户登录独立

---

## 2. 环境要求

- Node.js 20+
- PostgreSQL（本地或 Supabase）

---

## 3. 快速开始

```bash
cd gc-atelier
npm install
cp .env.example .env.local
# 编辑 .env.local
npm run dev
```

访问 http://localhost:3000

### 数据库初始化

```bash
npm run db:migrate   # 开发迁移
npm run db:seed      # 种子数据（含管理员）
```

**默认管理员（仅开发/首次 seed）：**

| 字段 | 值 |
|------|-----|
| 邮箱 | `admin@gcatelier.com` |
| 密码 | `admin123` |
| 入口 | `/admin/login` |

## 4. 本地环境变量

参考 `.env.example`：

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 |
| `NEXTAUTH_SECRET` | NextAuth 加密密钥 |
| `NEXTAUTH_URL` | `http://localhost:3000` |
| `STRIPE_SECRET_KEY` | `sk_test_...` |
| `STRIPE_WEBHOOK_SECRET` | Stripe CLI 转发所得 |
| `STRIPE_CURRENCY` | `usd` |
| `NEXT_PUBLIC_SHOP_CURRENCY` | `USD` |
| `USE_LOCAL_UPLOADS` | `true` 时写入 `public/uploads`（本地开发推荐） |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目 URL（生产上传必填） |
| `SUPABASE_SERVICE_KEY` | Supabase service_role key（生产上传必填） |

## 7. 认证架构

| 角色 | 入口 | 数据表 | 机制 |
|------|------|--------|------|
| 管理员 | `/admin/login` | `AdminUser` | `loginType: "admin"` |
| 客户 | 前台 LoginModal | `User` | `loginType: "customer"` |
| 旧链接 | `/login` | — | 重定向 `/admin/login` |

**后台权限：**

- `SUPER_ADMIN`：设置、用户管理
- `SALES`：不可访问 `/admin/users`

**关键文件：** `src/lib/auth.ts`、`src/middleware.ts`

## 8. 支付流程（开发视角）

```
登录 → 购物车 → 确认资料（session 只读）→ Pay securely
  → POST /api/payments/create-checkout-session
  → resolveCheckoutCustomer() 读 session
  → createPendingOrder() → Stripe Checkout
  → Webhook checkout.session.completed → fulfillPaidOrder()
  → /shop/checkout/success?session_id=...
```

**业务规则：**

- 结账必须登录
- 免运费：`shipping = 0`（`create-pending-order.ts`）
- 币种：`STRIPE_CURRENCY` + `NEXT_PUBLIC_SHOP_CURRENCY`

**关键 API：**

| 路径 | 用途 |
|------|------|
| `/api/payments/create-checkout-session` | 创建 Checkout |
| `/api/payments/webhook` | Stripe 回调 |
| `/api/payments/session` | 查询支付状态 |
| `/api/account` | 用户资料 |
| `/api/auth/register` | 注册 |

## 9. 图片上传

| 环境 | 存储 |
|------|------|
| 本地 | `public/uploads/`（`USE_LOCAL_UPLOADS=true`） |
| 生产（Vercel） | Supabase Storage（bucket: `uploads`） |

**关键文件：** `src/lib/blob-storage.ts`、`src/app/api/upload/route.ts`

本地开发用磁盘；Vercel serverless 无持久化磁盘，必须配置 Supabase Storage。

## 10. CMS 缓存机制

后台保存后调用 `revalidateSitePages()` 触发路径重验证。

**关键文件：** `src/lib/revalidate-site.ts`

生产缓存策略见 `next.config.mjs`（运维手册）。

---

## 11. 数据库变更规范

- **开发**：`npm run db:migrate`（Prisma migrate dev）
- **生产**：`npx prisma migrate deploy`（Vercel build 前或 Supabase SQL Editor）
- **禁止**：在生产 build 中执行 `prisma db push`

---

## 12. 开发常见问题

| 问题 | 处理 |
|------|------|
| 本地图片上传失败 | 设 `USE_LOCAL_UPLOADS=true` |
| Stripe 本地回调不到 | 用 Stripe CLI `listen` 转发 |
| 测试卡被拒 | 确认使用 `sk_test_` 密钥 |
| 结账未登录 | 设计如此，需先 Sign in |
| Prisma 客户端过期 | `npx prisma generate` |
